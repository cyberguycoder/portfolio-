EMS — Installation & Setup Guide

Home Project Setup

Step 1: System Requirements

✓ Operating System: Windows, macOS, or Linux
✓ Server: Apache 2.4 or above
✓ PHP Version: PHP 7.0 or above
✓ Database: MySQL 5.6 or above
✓ Browser: Modern browser (Chrome, Firefox, Safari, Edge)
📌 Note: It's recommended to use XAMPP or WAMP stack which includes all requirements in one package.

Step 2: Install XAMPP (Recommended)

1. Download XAMPP from https://www.apachefriends.org
2. Run the installer and follow the setup wizard
3. Choose installation directory and components (Apache, MySQL, PHP)
4. Complete installation and start Apache & MySQL services

Step 3: Extract Project Files

1. Navigate to XAMPP installation folder
2. Go to htdocs folder (usually C:\xampp\htdocs on Windows)
3. Extract EMS project files here
4. Folder structure should look like: htdocs/ems-project/
├── index.php
├── login.php
├── css/
├── js/
└── database/

Step 4: Database Setup

1. Open phpMyAdmin: http://localhost/phpmyadmin
2. Create new database named ems_db
3. Import SQL file:
- Select newly created database
- Click "Import" tab
- Browse and select ems_db.sql file
- Click "Go" to execute
⚠️ Database Name: Keep database name as ems_db or update config.php accordingly.

Step 5: Configure Database Connection

1. Open config.php file in the project root
2. Update database credentials: $db_host = "localhost";
$db_user = "root";
$db_password = "";
$db_name = "ems_db";
3. Save the file

Step 6: Access the Application

1. Start XAMPP Control Panel (Apache & MySQL should be running)
2. Open your browser and go to: http://localhost/ems-project
3. You should see the project homepage

Step 7: Admin Login

Username: admin
Password: admin123
🔒 Important: Change the default password immediately after first login for security.

Troubleshooting

Issue: "Cannot connect to database"
Solution: Check if MySQL service is running and verify database credentials in config.php
Issue: "404 Page Not Found"
Solution: Ensure project is in correct htdocs folder and correct folder name is used in URL
Issue: "Permission Denied"
Solution: Check file permissions; ensure files are readable by Apache server
